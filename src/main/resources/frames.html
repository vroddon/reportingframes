Reporting Frames Inventory

A domain-specific FrameNet inventory for legal reporting obligations

This tool annotates English legal provisions with a small, domain-specific inventory of FrameNet frames tailored to reporting obligations in European Union law. The general-purpose frames below are adapted so that their frame elements (FEs) align with the participants and temporal parameters characteristic of this domain: who must report what, to whom, and when. Each frame is serialised to its Framester URI, making every annotation interoperable linked data.

Annotation priority. For a given sentence the annotator returns up to three frames, ordered by relevance to reporting duties:
  1. Telling → Request → Receiving → Conditional_scenario → Time_period_of_action → Time_vector → Frequency → Calendric_unit
1

Telling

core reporting act Framester ↗

A Speaker (the obligated party) communicates a Message to an Addressee (typically a competent authority) — the core reporting act, evoked by verbs such as inform, notify, report, submit, disclose, communicate. Distinct from Request: the duty-bearer is transmitting information, not asking another party to act.

Core frame elements

Speaker Addressee Message

Non-core frame elements

Topic Medium Time Manner Means
2

Request

Framester ↗

A Speaker asks an Addressee or Recipient for something, or to carry out some action — e.g. an authority requiring a regulated entity to submit information (“the Authority may require …”).

Core frame elements

Speaker Addressee Recipient Message Message_argument Medium

Non-core frame elements

Manner Means Time Beneficiary Topic
3

Receiving

Framester ↗

A Recipient comes into possession of a Theme transferred from a Donor — e.g. an authority receives a report submitted by a regulated entity. The complement to Telling from the recipient's perspective.

Core frame elements

Donor Recipient Theme

Non-core frame elements

Means Manner Place Time Purpose_of_theme
4

Conditional_scenario

Framester ↗

A situation where two mutually exclusive possibilities are presented, each with a consequence. Typical of conditional reporting triggers (“if … then the entity shall report …”).

Core frame elements

Time_period_of_action

Time_vector

Framester ↗

An Event occurs at a particular Distance and Direction from a Landmark_event. Direction is often incorporated into the lexical unit (e.g. “ago”, “before”), used to express deadlines relative to a reference point.

Core frame elements

Direction Distance Event Landmark_event

Non-core frame elements

—

7

Frequency

Framester ↗

An Event recurs on a regular basis, characterized by how often it occurs over an Interval (e.g. “quarterly”, “annually”, “every month”) — the periodicity of recurring reporting obligations.

Core frame elements

Event Interval

Non-core frame elements

Attribute Degree Rate Salient_entity
8

Calendric_unit

Framester ↗

Names a conventional Unit of time within a calendric system (e.g. day, week, month, quarter, year), optionally positioned by Relative_time or within a larger Whole — granular temporal parameters of reporting requirements.

Core frame elements

Unit Name Relative_time

Non-core frame elements

Whole Salient_event
Frame definitions are the authoritative inventory used by this tool's annotator (ClaudeClient system prompt). Frame URIs resolve into the Framester linked data hub. Annotations are exported as Turtle RDF using the NIF standard.
